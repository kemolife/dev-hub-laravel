DevHub Design

Visual specifications for the Laravel build. Each mockup is a static HTML file using the same design tokens that will eventually become the Tailwind config.

01 — Homepage feed

The first impression. Reading time foregrounded, no engagement counters in the feed.

02 — Post detail

The reading experience. Serif body, narrow column, reactions hidden until end of read.

03 — Post editor

The writing experience. Distraction-free with depth signals (word count, reading time, code blocks).

Design principles

  1. Calm over urgent — no badges, no red dots, no streaks
  2. Depth over breadth — reading time and word count are first-class signals
  3. Author over algorithm — chronological feed, no "for you"
  4. Whitespace is a feature — generous margins, no cramped UI
  5. Sentence case always — never Title Case, never ALL CAPS

Design tokens

All defined in assets/styles.css as CSS variables. When porting to Laravel, these become Tailwind theme.extend values.
